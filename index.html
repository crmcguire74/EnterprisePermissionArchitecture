Overview

Enterprise Permission Architecture Redesign

A comprehensive approach to organizing permissions, roles, and licenses within a financial services firm.

Current Challenges

  • Individual permissions assigned directly to users
  • Chaotic onboarding processes
  • Difficulty supporting workflows during staff absences
  • Lack of structure in permission management
  • Complex license management across third-party integrations
  • Migration from Active Directory to Azure Entra ID

Our Solution

We propose a comprehensive role-based access control (RBAC) framework integrated with Microsoft Entra ID that:

  • Organizes permissions by business function, role, and workflow
  • Streamlines onboarding through role templates
  • Provides clear license management and visibility
  • Enables seamless workflow support during absences
  • Creates a scalable, maintainable permission architecture

Role-Based Access Control (RBAC) Structure

The new permission architecture is designed around a hierarchical RBAC model with the following key components:

1. Business Functions

Top-level organizational units corresponding to major business areas (Finance, Trading, Operations, IT, Compliance, etc.).

2. Roles

Standardized job functions within each business area. Roles inherit permissions from their parent business function and add specific permissions needed for their responsibilities.

3. Permission Sets

Collections of specific permissions grouped by system, application, or resource type. These are attached to roles rather than directly to users.

4. Application Access Templates

Pre-defined configurations for specific applications that define levels of access (view, edit, admin) and required license types.

5. Data Access Controls

Permissions specific to databases, file systems, and data resources, organized by sensitivity level and business need.

Microsoft Entra ID Integration

Microsoft Entra ID (formerly Azure AD) provides the foundation for our new permission architecture:

  • Administrative Units: Map to business functions for delegated administration
  • Security Groups: Represent roles and permission sets
  • Dynamic Groups: Automatically assign users based on attributes
  • Privileged Identity Management (PIM): For just-in-time privileged access
  • Conditional Access Policies: Context-aware security controls
  • Entitlement Management: For access package creation and approval workflows

This structure leverages Entra ID's capabilities while providing a clear, logical organization that maps to the firm's business processes.

Migration Approach

Transitioning from the current permission model to the new RBAC framework requires a phased approach:

1

Discovery & Analysis

  • Inventory existing AD groups and permissions
  • Map current users to business functions
  • Document application access requirements
  • Identify license dependencies
2

Design & Planning

  • Define business functions and roles
  • Create permission sets
  • Design Entra ID structure
  • Develop migration timeline
3

Pilot Implementation

  • Set up Entra ID structure
  • Migrate a small department/team
  • Test and validate access
  • Gather feedback and refine
4

Staged Rollout

  • Department-by-department migration
  • Run parallel systems during transition
  • Validate permissions post-migration
  • Provide training and support
5

Decommissioning

  • Retire legacy AD groups
  • Document new architecture
  • Implement ongoing governance
  • Establish review processes

Migration Best Practices

Role Mining & Analysis

Use automated tools to analyze existing permission patterns and identify natural role groupings before creating new role definitions.

Least Privilege Principle

Use the migration as an opportunity to implement least privilege access, removing unnecessary permissions during role creation.

Stakeholder Engagement

Involve business unit leaders in defining roles and required access to ensure business continuity during migration.

Documentation & Training

Create comprehensive documentation for the new permission model and provide training for IT staff and end users.

Monitoring & Metrics

Establish KPIs to measure the success of the migration, such as reduced onboarding time, decreased support tickets, and improved audit outcomes.

Migration Challenges & Mitigations

Risk: Users losing access during migration

Mitigation: Implement parallel access during transition, schedule migrations during low-activity periods, and provide emergency access mechanisms.

Risk: Missing critical permissions during role creation

Mitigation: Use automated discovery tools, involve business users in validation, and implement a rapid remediation process for missed permissions.

Risk: Resistance to new permission model and processes

Mitigation: Clear communication about benefits, comprehensive training, and showcasing early wins from pilot implementations.

Risk: Disruption to third-party application access

Mitigation: Map license dependencies early, involve vendors in transition planning, and test license activation/deactivation during pilot phase.

License Management Strategy

Our redesigned license management framework integrates with the RBAC permission model to provide comprehensive visibility and control over third-party software licenses.

Key Components:

  • License Registry: Centralized repository of all software licenses, including terms, costs, and renewal dates
  • License Types: Categorization by user-based, application-based, or concurrent usage models
  • License Assignment: Direct mapping between licenses, applications, and roles
  • Usage Monitoring: Tracking utilization to optimize license allocation
  • Automated Provisioning: License assignment based on role changes
  • Communication Framework: Targeted notifications to affected users and stakeholders

License Categories:

User-Based Licenses

Assigned directly to individuals (e.g., Bloomberg Terminal, Reuters Eikon)

Application-Based Licenses

Assigned to internal applications that integrate third-party services

Concurrent-Use Licenses

Shared licenses with limited simultaneous users

Enterprise Licenses

Unlimited usage across the organization

License Communication Framework

The integrated communication system ensures stakeholders receive timely information about licenses:

Renewal Notifications

Automated alerts to stakeholders before license renewals, including cost implications and usage statistics

Maintenance Windows

Notifications about planned service outages or updates from vendors

Compliance Alerts

Warnings about potential licensing compliance issues or usage thresholds

Usage Reports

Regular utilization reports to business owners for optimization decisions

Entra ID License Integration

Our solution leverages Microsoft Entra ID's license management capabilities:

Microsoft License Management

Direct management of Microsoft licenses through Entra ID:

  • Group-based license assignment aligned with roles
  • Automatic license provisioning/deprovisioning
  • License usage monitoring and reporting
  • Service plan customization by role

Third-Party License Integration

For non-Microsoft applications:

  • Custom attributes in Entra ID to track license entitlements
  • Integration with third-party license management systems via APIs
  • Automated provisioning using Entra ID's SCIM connectors
  • License mapping to roles and groups in Entra ID

License Optimization Strategies

License Pooling

Create pools of shared licenses for applications with intermittent usage, allowing more users to be supported with fewer licenses.

Usage-Based Reclamation

Automatically reclaim licenses from users who haven't accessed the application within a defined period.

Tiered Access Models

Implement differentiated license types (basic, standard, premium) based on role requirements to reduce costs.

Application Consolidation

Identify overlapping functionality in licensed applications and consolidate to reduce total license requirements.
